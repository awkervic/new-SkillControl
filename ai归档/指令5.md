## 2. 全新多仓库与技能管理联动机制要求

### 功能项 A：独立的「仓库管理」模块
1. **添加仓库：** 界面上提供一个“添加仓库”表单，用户可以输入“仓库名称”和“Git 仓库链接（如 GitHub 链接）”。点击保存后，Rust 后端全自动为其生成唯一的 `repo_id` 并追加写入到 `config.json` 的 `repositories` 数组中。
2. **仓库列表看盘：** 在设置或独立侧边栏中，整齐列出用户当前已绑定的所有仓库地标，并支持一键删除仓库（删除仓库时，自动解除该仓库下所有技能的绑定状态）。

### 功能项 B：带筛选与搜索的「技能管理」主看板
1. **按仓库筛选（Filter by Repo）：** 前端主页提供一个干净的侧边栏菜单（或顶部下拉筛选框），里面列出【全部仓库】以及每一个已添加的【具体仓库名称】。点击某个仓库，界面下方的技能花名册必须瞬间秒级过滤，只展示隶属于该仓库的 Skills。
2. **全局跨仓库搜索：** 搜索框支持高频模糊检索。用户输入关键词时，系统会同时在所有已加仓库的技能标题和描述中进行匹配，不管它属于哪个仓库。
3. **远程技能树发现与按需拉取：** - 当用户点击某个仓库或刷新时，Rust 后端去异步扫描/解析该远程 Git 仓库中的技能清单。
   - 前端卡片展示这些“云端发现”的技能。当用户真正点击某个技能卡片上的“下载/安装”时，Rust 后端才真正执行 Git 动作，将对应的 `.md` 剧本拉取到本地 Staging 暂存池（`./my-brain/`）中。

## 3. 其他核心功能模块要求

### 功能模块 C：每个技能卡片的“铁血三连控制开关”
在前端的技能列表中，每一个技能必须通过修改 `config.json` 中的 `skills_status` 节点，来严格联动执行物理动作：
1. **「启用开关」（AGY 与 Reasonix 独立双向控制）：**
   - 点亮 **AGY 启用** 时：自动在全局隐藏路径 `C:\Users\<当前用户名>\.gemini\skills\shared\<技能ID>\` 下创建文件夹，并将技能拷贝为 `SKILL.md`。
   - 点亮 **Reasonix 启用** 时：直接将该裸 `.md` 文件拷贝到 Reasonix 的隐藏共享路径 `C:\Users\<当前用户名>\.reasonix\playbooks\shared\<技能ID>.md` 下。
   - 关闭开关时：物理删除/移除上述对应路径下的分发文件，让技能完全下线。
2. **「更新开关（自动更新）」：** 标记该技能是否随软件启动自动执行 `git pull`。
3. **「手动更新按钮（Sync Now）」：** 立刻为这一个技能触发后台的 Git 异步拉取与本地物理文件分发。

### 功能模块 D：全局设置面板 (Settings Panel)
- **主题换肤：** 支持一键切换日间（Light）/夜间（Dark）模式，通过修改 CSS Variables 变量实现，状态记录在账本的 `theme` 字段。
- **WebDAV 配置与自动备份：** 提供 WebDAV 完整的账号、密码、服务器地址表单。如果 `auto_backup_enabled` 开关被点亮，每次应用启动或技能启用状态发生变更时，大后方全自动执行无感静默备份。

### 功能模块 E：WebDAV 云端声明式备份
- **纯配置压缩备份：** 点击备份时，Rust 后端不需要备份庞大的实体文件，而是直接把包含所有仓库列表、技能启用状态、WebDAV 设置、主题颜色的 `config.json` 账本文件，压缩打包成 `config_backup.zip` 并通过 HTTP PUT 上传。
- **一键复活：** 支持从云端下载该压缩包还原 `config.json`。还原后，软件根据账本内容全自动、按需重新克隆 Git 仓库并物理重组分发文件，实现整个工作流在新电脑上的一键完美复活！
